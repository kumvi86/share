# 8.12 비대칭 타일링
문제) 2xn 크기의 직사각형을 2x1 크기의 타일로 채우자
타일들은 서로 겹쳐서는 안 되고 90도로 회전해서 쓸 수 있다.
단, 좌우 대칭이어서는 안 된다.

풀이) 
완전 탐색의 함정 = 전체 타일링 갯수 - 대칭 타일링 갯수 (= 비대칭 타일링 갯수)
대칭만 세는 게 쉽다!
그림 8.9는 대칭 타일링 방법이다.
n이 홀수이면 가운데 세로 타일이 들어가고(예로 n을 5라고 생각하면 2-1-2)
n이 짝수이면 가운데 가로 타일이 들어가 2개가 빠진다. (예로 n이 6이라면 2-2-2 or 3-3)
-> 결국 대칭은 가운데를 뺀 회색 부분을 채운 경우의 수와 동일하다.


코드 8.19
if(width %2 == 1) // 얘는 홀수의 경우
    return (tiling(width) - tiling(width/2) + MOD)%MOD
    -> 전체에서 대칭 케이스를 뺀 거이다. width가 5면 전체에서 width가 2인 타일링 방법의 경우의 수를 뺀 거이다.

나머지 아래 코드는 짝수에 대한 경우으로 width가 6이면 비대칭 타일링 방법의 수는 전체에서 width가 3인 방법의 수와 width가 2인 방법의 수를 제외한 것이다.

MOD를 더한 이유는? tiling이 MOD로 나눈 나머지를 반환하기 때문이다. 

직접 비대칭 타일링의 수 세기
그림 8.10처럼 비대칭의 케이스를 나누자.
a,b는 회색이 비대칭이어야 한다. c,d는 채우기만 하면 된다.
코드 8.20에서 tiling은 주어진 구간에서 타일을 채우는 모든 경우의 수를 나타내므로 c, d는 tiling을 이용해서
a,b는 비대칭이 아닌 것만 세어야 하므로 재귀호출로 더 쪼개면서 찾아가는 것이다.

# 8.14 폴리오미노
폴리오미노란? 정사각형의 변들을 서로 완전하게 붙여 만든 도형
문제) 세로로 단조인 폴리오미노의 수가 몇 개이냐?
가로줄을 그었을 때 정사각형들이 연결되어 있어야 한다.
그래서 b가 세로 단조 폴리오미노가 아니다.

풀이)
n개의 정사각형으로 구성된 모든 폴리오미노들은 첫 번째 가로줄에 포함된 정사각형의 수를 기준으로 분류합시다.
첫 번째 가로줄에 1개의 정사각형이 있는 경우, 2개의 정사각형, 3개, 4개 이런 식으로..
그러나 이렇게 단순하게 할 수가 없다.
왜냐하면 첫 번째줄과 두 번째줄이 어떻게 붙이냐에 따라서 경우의 수가 달라지기 때문이다.
그게 267 페이지의 그림이 나타나는 것

//좌변:전체 n개의 정사각형 중 첫 줄에 first개의 정사각형이 들어있는 폴리오미노의 수
//우변: 남은 정사각형(=n-first) 중에서 첫 번째 줄이 1개부터 시작해서 n-first개까지를 더한 경우의 수
poly(n, first) = poly(n-first, 1) + poly(n-first, 2) + poly(n-first, n-first)

-> 이제 여기에 붙이는 방법에 대해서 적용하자.
페이지 268에 나오는 그림이 붙이는 방법에 대한 설명으로 첫 줄에 2개가 있고 두 번째 줄에 3개가 있으면 붙일 수 있는 방법의 수 2+3-1 =4개이다.
우변에 적용하면
(first+1-1)*poly(n-first,1) + (first+2-1)*poly(n-first,2) + ... + (first+n-first-1)*poly(n-first, n-first)
= first*poly(n-first, 1) + (first+1)*poly(n-first,2) + ... +(n-1)*poly(n-first, n-first)
시그마까지 붙여가며 정리하면 된다.
